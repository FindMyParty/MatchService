## Codebase Patterns
- Project uses pure ESM modules (`"type": "module"` in package.json)
- Hexagonal architecture: src/domain/, src/application/, src/adapters/inbound/, src/adapters/outbound/
- Always use `IF NOT EXISTS` for migrations (idempotence requirement)
- Table name for match is `match_tb` (not `match`) to avoid SQL reserved word conflict
- Composite PK in match_tb requires raw SQL (kysely doesn't support it natively)
- Use `.gitkeep` for empty directories until story populates them
- vitest.config.js sets `passWithNoTests: true` — tests won't fail when no test files exist yet
- All environment access is centralized in `src/config/env.js`; other modules import `env` from there
- shared/ modules (logger.js, telemetry.js) read process.env directly — they CANNOT import from config/env.js
- OTel NodeSDK uses `metricReader` (singular) not `metricReaders` in sdk-node 0.55.0
- Sentry init goes in main.js (US-016), NOT in telemetry.js — telemetry.js only handles OTel SDK
